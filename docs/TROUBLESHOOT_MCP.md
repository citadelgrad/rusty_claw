# MCP Integration Troubleshooting

This guide covers common issues when using SDK MCP servers (in-process Rust tools) with rusty_claw.

---

## The CLI hangs and produces no output

**Symptom:** The `cargo run` command starts but never prints any response. No error is shown.

**Cause:** The Claude CLI hangs before startup when it receives unexpected arguments or an invalid configuration.

**Solutions:**

1. Do not pass SDK server configs via `--mcp-config`. The SDK handles this automatically by registering SDK servers in the control protocol `initialize` request. The `SdkMcpServer` config in `ClaudeAgentOptions.sdk_mcp_servers` is the correct place.

2. Make sure you call `client.register_mcp_message_handler(Arc::new(registry)).await` before `client.connect()`, not after.

3. Verify the server name in `SdkMcpServer` exactly matches the name used in `SdkMcpServerImpl::new(...)` and `SdkMcpServerRegistry::register(...)`.

```rust
// Correct: names must match across all three places
let options = ClaudeAgentOptions::builder()
    .sdk_mcp_servers(vec![SdkMcpServer {
        name: "my_server".to_string(),  // <-- must match below
        version: "1.0.0".to_string(),
    }])
    .build();

let mut server = SdkMcpServerImpl::new("my_server", "1.0.0");  // <-- same name
server.register_tool(my_tool());

let mut registry = SdkMcpServerRegistry::new();
registry.register(server);  // <-- registers under "my_server"

let mut client = ClaudeClient::new(options)?;
client.register_mcp_message_handler(Arc::new(registry)).await;  // before connect()
client.connect().await?;
```

## Claude calls the wrong tool or a tool is not found

**Symptom:** Claude reports it cannot find `mcp__my_server__my_tool`, or calls a different tool than expected.

**Cause:** Tool naming follows a strict convention. Claude sees MCP tools as `mcp__{server_name}__{tool_name}` with double underscores.

**Solutions:**

1. Check that the tool name passed to `SdkMcpTool::new()` (or generated by `#[claw_tool]`) exactly matches what you expect Claude to call.

2. Include the fully qualified name in `allowed_tools`:

```rust
let options = ClaudeAgentOptions::builder()
    .sdk_mcp_servers(vec![SdkMcpServer {
        name: "text_tools".to_string(),
        version: "1.0.0".to_string(),
    }])
    .allowed_tools(vec![
        "mcp__text_tools__word_count".to_string(),
        "mcp__text_tools__repeat".to_string(),
    ])
    .build();
```

3. To discover what names the CLI assigned, inspect the `SystemMessage::Init` message. The `tools` field lists all available tool names including MCP tools.

## Tool is called but returns an error

**Symptom:** Claude receives a tool result with `is_error: true` or a JSON-RPC error response.

**Cause:** The tool handler returned `Err(ClawError)` or `ToolResult::error(...)`.

**Debugging steps:**

1. Add logging to your tool handler:

```rust
#[async_trait]
impl ToolHandler for MyHandler {
    async fn call(&self, args: Value) -> Result<ToolResult, ClawError> {
        eprintln!("Tool called with args: {}", args);
        // ... your logic
    }
}
```

2. Check JSON-RPC error codes:
   - `-32601` — Method not found (the server received an unknown JSON-RPC method)
   - `-32602` — Tool not found by name (check that `register_tool` was called before `connect`)
   - `-32603` — Tool handler returned `Err(ClawError)` (check the error message)

3. Make sure required parameters have the correct types. The `#[claw_tool]` macro generates parameter extraction that returns `ClawError::ToolExecution` for missing or mistyped required fields.

## `#[claw_tool]` macro generates unexpected tool name

**Symptom:** The tool name seen by Claude differs from the function name.

**Behavior:** By default, `#[claw_tool]` converts the function name to kebab-case (replaces `_` with `-`). For example, `fn word_count(...)` generates a tool named `word-count`.

**Solution:** Specify the name explicitly to avoid surprises:

```rust
#[claw_tool(name = "word_count", description = "Count words in text")]
async fn word_count(text: String) -> ToolResult {
    ToolResult::text(format!("{}", text.split_whitespace().count()))
}
```

## Tool receives `null` or wrong argument types

**Symptom:** `args["param"].as_str()` returns `None`, or numeric conversions fail.

**Cause:** Claude may send arguments as different JSON types than expected.

**Solutions:**

1. Use the `#[claw_tool]` macro — it generates type-checked parameter extraction and produces clear `ClawError::ToolExecution` errors when types don't match.

2. When implementing `ToolHandler` manually, add defensive checks:

```rust
async fn call(&self, args: Value) -> Result<ToolResult, ClawError> {
    let text = args["text"]
        .as_str()
        .ok_or_else(|| ClawError::ToolExecution("missing required 'text' parameter".into()))?;
    // ...
}
```

## MCP server is not listed in the system init message

**Symptom:** `SystemMessage::Init` arrives but the `mcp_servers` field does not include your SDK server.

**Cause:** The server was registered too late (after `connect()`), or the `sdk_mcp_servers` option was not set.

**Solution:** Both the options config and the handler registration must happen before `connect()`:

```rust
// 1. Configure options
let options = ClaudeAgentOptions::builder()
    .sdk_mcp_servers(vec![SdkMcpServer { name: "my_server".into(), version: "1.0.0".into() }])
    .build();

// 2. Build client
let mut client = ClaudeClient::new(options)?;

// 3. Register handler BEFORE connect
client.register_mcp_message_handler(Arc::new(registry)).await;

// 4. Connect
client.connect().await?;
```
